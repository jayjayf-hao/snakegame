# 贪吃蛇游戏（Snake Game）

一个基于 Windows 控制台的模块化贪吃蛇游戏，采用双缓冲渲染技术，支持游戏存档/读档功能，使用 CMake 构建系统组织项目。

## 功能特性

- **双缓冲渲染**：使用两个控制台屏幕缓冲区，实现无闪烁的画面更新。
- **模块化设计**：代码分离为独立的类（Snake、Food、Renderer、Input、SaveSystem、Game）。
- **存档/读档系统**：支持将游戏状态保存到文本文件，并随时加载恢复。
- **游戏状态管理**：支持菜单、游戏进行、暂停、游戏结束等多种状态。
- **分数系统**：每吃到一个食物得 10 分。
- **游戏信息面板**：在游戏窗口右侧显示控制说明和游戏状态。

## 游戏控制

| 按键 | 功能 |
|------|------|
| **W/A/S/D** | 上下左右移动蛇 |
| **P** | 暂停/继续游戏 |
| **C** | 保存游戏到 `save.txt` |
| **V** | 从 `save.txt` 加载游戏 |
| **X** | 退出游戏 |

## 编译和运行

### 编译步骤

```powershell
# 进入项目目录
cd d:\snake\SnakeGame

# 创建构建目录
mkdir build
cd build

# 使用 CMake 配置项目
cmake ..

# 编译项目
cmake --build .
```

### 运行游戏

```powershell
# 从 build 目录中运行
.\Debug\SnakeGame.exe
```

## 核心模块说明

### Point.h / Point.cpp
定义游戏中的坐标点结构，用于表示蛇身节点和食物位置。

### Direction.h
定义游戏中的移动方向枚举（停止、左、右、上、下）。

### Snake 类
管理贪吃蛇的身体和移动：
- 使用 `std::deque<Point>` 存储蛇身各节点
- 支持移动、生长、碰撞检测
- 防止蛇头反向进入蛇身

### Food 类
管理游戏中的食物：
- 随机生成食物位置
- 确保食物不会生成在蛇身上

### Renderer 类（双缓冲核心）
实现双缓冲渲染技术：
- 使用两个 Windows 控制台屏幕缓冲区
- 在后台缓冲区构建画面
- 一次性输出并交换缓冲区，避免闪烁

### Input 类
处理键盘输入：
- 支持 WASD 方向键、特殊功能键
- 非阻塞式读取用户输入

### SaveSystem 类
实现游戏状态的序列化和反序列化：
- 存储格式：文本文件（.txt）
- 包含：蛇身位置、移动方向、食物位置、分数
- 文件格式示例：
  ```
  score=100
  direction=1
  food=15,12
  snake=10,10;10,11;10,12;11,12
  ```

### Game 类
游戏主类，管理整体游戏流程：
- 初始化游戏对象（蛇、食物、渲染器）
- 游戏主循环
- 状态管理和更新
- 碰撞检测

**遇到的错误与调试过程**：
1. **编译错误**：结构体类型重定义，通过将 GameState 改名为 SaveData 并分离到不同的头文件解决
2. **链接错误**：缓冲区被占用，通过关闭正在运行的进程解决
3. **渲染问题**：分数显示不出来，通过扩大窗口缓冲区并改进 drawGameInfo 函数解决
4. **游戏逻辑**：通过完善蛇的初始状态和食物生成算法解决潜在的边界碰撞问题

## 技术特点

1. **模块化架构**：每个功能独立为一个类，便于维护和扩展
2. **数据结构**：采用 `std::deque` 管理蛇身，支持高效的首尾操作
3. **双缓冲技术**：避免屏幕闪烁，提供流畅的游戏体验
4. **状态模式**：使用枚举管理游戏状态
5. **文件持久化**：支持游戏进度保存和读取
